# MSA6 이미지 측정 / 불량 감지 - 기술 정확도 분석 및 기능 제안

## 1. 선 측정 (Line Measurement) - 현재 로직 분석 및 개선점

### 1.1 현재 로직 흐름

```
사용자 선 그리기 → 3000포인트 샘플링 → 밝기 계산(R채널만)
→ 이동평균(window=5) 스무딩 → 밝기 전환점 탐지(단일 임계값)
→ 세그먼트 분할 → 비례 길이값 계산 → 보정(마지막 세그먼트에만)
```

### 1.2 기술 정확도 개선점

#### (A) 밝기 계산: R채널 단독 사용 문제
- **현재**: `msa6_useBrightness.js:51` - `imageData.data[index]` (R채널만 사용)
- **문제**: 그레이스케일 이미지에서는 R=G=B이므로 괜찮지만, 컬러 이미지나 약간의 색조 차이가 있는 이미지에서는 부정확
- **제안**: Luminance 공식 적용
  ```js
  // 현재
  totalBrightness += data[index]; // R만

  // 개선: ITU-R BT.601 Luminance
  const r = data[index];
  const g = data[index + 1];
  const b = data[index + 2];
  totalBrightness += 0.299 * r + 0.587 * g + 0.114 * b;
  ```
- **영향도**: 낮음 (현재 SEM/현미경 이미지는 대부분 그레이스케일이므로). 하지만 범용성을 위해 적용 권장

#### (B) 밝기 전환점 탐지: 단일 임계값의 한계
- **현재**: `msa6_measures.js:161` - `point.brightness > this.brightnessThreshold` (하드 바이너리 비교)
- **문제**: 임계값 근처에서 노이즈로 인한 빠른 false 전환 발생 → 미세한 세그먼트 다수 생성
- **제안 1: 히스테리시스 (이중 임계값)**
  ```
  밝음→어두움 전환: brightness < threshold - margin
  어두움→밝음 전환: brightness > threshold + margin
  margin = threshold의 5~10%
  ```
  이렇게 하면 임계값 근처의 노이즈에서 불필요한 전환이 발생하지 않음

- **제안 2: 최소 세그먼트 길이 필터링**
  ```
  전환점 사이의 거리가 전체 라인의 1% 미만이면 무시
  ```
  매우 짧은 세그먼트는 노이즈일 가능성이 높으므로 제거

#### (C) 이동평균 윈도우 크기 고정 문제
- **현재**: `msa6_measures.js:146` - `windowSize = 5` (3000 샘플 중 5개)
- **문제**: 이미지 해상도나 측정선 길이에 관계없이 동일한 윈도우 사용. 고해상도 이미지에서는 노이즈 필터링 부족, 저해상도에서는 과도한 스무딩
- **제안**: 적응형 윈도우 크기
  ```js
  // 측정선의 실제 픽셀 길이에 비례
  const pixelLength = Math.sqrt(
    Math.pow(end.x - start.x, 2) + Math.pow(end.y - start.y, 2)
  );
  const windowSize = Math.max(3, Math.min(21, Math.round(pixelLength / 100)));
  // 최소 3, 최대 21 (홀수 유지)
  ```

#### (D) 세그먼트 값 보정 방식
- **현재**: `msa6_measures.js:313-318` - 전체 값과 세그먼트 합의 차이를 마지막 세그먼트에만 보정
- **문제**: 마지막 세그먼트에 오차가 집중됨
- **제안**: 비례 분배 보정
  ```js
  const correction = totalValue - segmentsSum;
  segments.forEach(seg => {
    seg.value += correction * (seg.value / segmentsSum);
  });
  ```

#### (E) 서브픽셀 보간 미적용
- **현재**: `calculateBrightness`에서 `Math.floor`로 정수 좌표만 사용
- **문제**: 픽셀 경계에서의 밝기값이 불연속적 → 전환점 위치 정확도 저하
- **제안**: 바이리니어 보간(Bilinear Interpolation) 적용
  ```js
  // 소수점 좌표에서의 밝기를 주변 4개 픽셀의 가중평균으로 계산
  const fx = imageX - Math.floor(imageX);
  const fy = imageY - Math.floor(imageY);
  brightness = (1-fx)*(1-fy)*p00 + fx*(1-fy)*p10 + (1-fx)*fy*p01 + fx*fy*p11;
  ```
  특히 스케일바 기반 고정밀 측정에서 중요

## 2. 불량 감지 (Defect Detection) - 현재 로직 분석 및 개선점

### 2.1 현재 알고리즘: 박스 수축 방식

```
사용자 영역 선택 → 사각 박스 설정 → 1px씩 안쪽으로 수축
→ 각 단계에서 경계 픽셀 임계값 검사 → 멈춘 픽셀 기록
→ Union-Find 8방향 클러스터링 → 클러스터 병합(5px 이내)
→ 타원 피팅(최대거리 기반) → Striation/Distortion 계산
```

### 2.2 기술 정확도 개선점

#### (A) 박스 수축 알고리즘의 근본적 한계
- **문제 1: 내부 불량 미감지** - 박스 경계가 지나가지 않는 내부의 작은 불량은 감지 불가
  ```
  예시: 영역 중앙에 있는 작은 결함이 박스 수축 경계에 도달하기 전에
  배경 종료 조건에 의해 감지가 중단될 수 있음
  ```
- **문제 2: 직사각형 스캔 편향** - 원형 불량도 직사각형 경계 방향으로 편향된 형태로 감지됨
- **제안: Connected Component Labeling (CCL) 병행**
  ```
  1단계: 전체 영역에서 임계값 기반 이진화
  2단계: CCL로 연결된 영역 추출
  3단계: 기존 박스 수축 결과와 CCL 결과를 병합
  ```
  CCL은 영역 전체를 한 번에 스캔하므로 내부 불량도 놓치지 않음

#### (B) 배경 종료 조건의 매직 넘버
- **현재**:
  - `backgroundDetectionThreshold = 0.8` (80%)
  - `maxConsecutiveHighDetection = 15`
  - `minBoxShrinkSteps = 20`
- **문제**: 이미지 특성에 따라 적절한 값이 다름. 현재는 하드코딩
- **제안**: 이미지 통계 기반 자동 조정
  ```js
  // 선택 영역의 밝기 분포 분석
  const { mean, stddev } = analyzeRegionStatistics(boundingBox);

  // 대비가 낮은 이미지 → 더 보수적 종료 조건
  if (stddev < 30) {
    this.backgroundDetectionThreshold = 0.9;
    this.maxConsecutiveHighDetection = 20;
  }
  // 대비가 높은 이미지 → 더 공격적 종료 조건
  else if (stddev > 80) {
    this.backgroundDetectionThreshold = 0.7;
    this.maxConsecutiveHighDetection = 10;
  }
  ```

#### (C) 타원 피팅 정확도
- **현재**: `calculateAccurateEllipseRadius` - 중심으로부터 최대 거리 + 여유값 사용
- **문제**: 실제 불량의 기울어진 타원이나 비대칭 형태를 제대로 반영하지 못함
- **제안: PCA 기반 타원 피팅**
  ```
  1. 클러스터 픽셀들의 공분산 행렬 계산
  2. 고유값/고유벡터로 주축 방향과 크기 결정
  3. 타원의 회전 각도까지 정확하게 산출
  ```
  이렇게 하면 기울어진 불량도 정확하게 타원으로 표현 가능

#### (D) Striation 계산의 정밀도
- **현재**: 8-bin 방향 히스토그램 + 엔트로피 기반
- **문제**: 8방향은 해상도가 매우 낮음. 45도 단위로만 구분 가능
- **제안**:
  ```
  - 방향 빈 수를 16~32개로 증가
  - 또는 Radon Transform 기반 방향성 분석 (더 정확하지만 계산량 증가)
  - 공간 주파수 분석 (FFT 기반) 추가 옵션
  ```

#### (E) 클러스터 병합 거리의 고정값 문제
- **현재**: `areClustersAdjacentImproved` - 5픽셀, 3픽셀 고정 거리
- **문제**: 고해상도 이미지에서는 5픽셀이 너무 작고, 저해상도에서는 너무 큼
- **제안**: 이미지 해상도 및 불량 크기에 비례하는 거리
  ```js
  const mergeDistance = Math.max(3, Math.round(
    Math.sqrt(selectedAreaWidth * selectedAreaHeight) * 0.02
  ));
  ```

#### (F) 임계값 자동 추천 기능
- **현재**: 사용자가 수동으로 밝기 임계값(brightnessThreshold=200)을 설정
- **문제**: 이미지마다 최적 임계값이 다르며, 사용자가 시행착오로 찾아야 함
- **제안: Otsu's Method 적용**
  ```
  선택 영역의 밝기 히스토그램을 분석하여
  클래스 간 분산을 최대화하는 임계값을 자동으로 계산
  → "자동 임계값 추천" 버튼으로 사용자에게 제안
  ```

## 4. 코드 레벨 주요 발견사항

### 4.1 `getPixelBrightnessAccurate` (불량 감지용)
- 파일: `msa6_defect_detection.js:1571-1591`
- R채널만 사용 (선 측정의 `calculateBrightness`와 동일한 패턴)
- 원본 이미지 데이터와 캔버스 좌표 변환은 정확하게 구현됨

### 4.2 `createBoundedSegments` (선 측정 세그먼트 분할)
- 파일: `msa6_measures.js:126-339`
- `this` 컨텍스트를 `call`로 전달하는 패턴 사용 (`_buildMeasurementContext`에서 plain object 생성)
- `getDistanceRatio` 함수가 별도 composable(`msa6_useGeometry.js:85`)에 정의되어 있으나 `this` context에 포함되지 않아 런타임 에러 가능성 있음 → 확인 필요

### 4.3 불량 감지 성능
- `setTimeout(2ms)` 기반 비동기 루프 → Web Worker로 이동하면 UI 블로킹 완전 제거 가능
- 대용량 픽셀(10,000+)에서 공간 분할(4x4 그리드) 사용 → 적절한 최적화

### 4.4 스케일바 관리
- localStorage에 이미지별 키로 저장, 최대 50개 프루닝 → 적절
- 자동 감지 기능이 실제 구현되지 않음 (`scaleBarDetected = false`로 하드코딩) → 향후 OCR/패턴 매칭으로 구현 가능

## 5. 개선 우선순위 요약

### 즉시 적용 가능 (난이도 낮음, 효과 높음)
1. **히스테리시스 전환점 탐지** - false 세그먼트 방지
2. **적응형 이동평균 윈도우** - 이미지 해상도에 맞는 스무딩
3. **측정 결과 CSV 내보내기** - 데이터 활용성
4. **Shift 키 수평/수직 스냅** - 정확한 측정 보조

### 중기 개선 (난이도 중간, 효과 높음)
5. **Otsu 자동 임계값 추천** - 사용자 시행착오 감소
6. **밝기 프로파일 그래프** - 측정 검증 도구
7. **CCL 기반 불량 감지 병행** - 내부 불량 미감지 해결
8. **PCA 기반 타원 피팅** - 불량 형태 정확도 향상

### 장기 개선 (난이도 높음)
9. **Canny 에지 검출 모드** - 근본적 측정 정확도 향상
10. **Web Worker 불량 감지** - UI 반응성 개선
11. **스케일바 자동 감지 (OCR)** - 사용 편의성
